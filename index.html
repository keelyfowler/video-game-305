How to Use GameVault

This guide walks you through everything you need to get started with GameVault — no technical experience required.

⚡ Golden Path — the full GameVault experience
Sign Up → Create Profile → Search for a Game → View Game Details → Write a Review → View it on the Feed → Follow Another User
🔍 Search & Game Info
1
Open the app and tap the Discover tab at the bottom of the screen.
2
Type any game name into the search bar and tap the search button or press enter on your keyboard.
3
A list of matching games appears. Each result shows the game title, cover art, and basic info pulled from the IGDB database.
4
Tap any game to open its detail page. Here you can see the genre, platform, developer, screenshots, and full description.
5
From the game detail page, tap Write a Review to leave your rating, or See All Reviews to read what the community thinks.
👤 Onboarding & Profile
1
When you open GameVault for the first time, you'll see the sign in / sign up screen. Use the toggle at the top to switch between Sign In (returning user) and Sign Up (new account).
2
To create a new account, select Sign Up, enter your email address and a password, then tap Sign Up.
3
You'll be taken to the Create Your Gamer Profile screen. Fill in your display name, username, bio, and optional fields like favorite genres, Discord handle, and Twitch handle.
4
If you want to set up your profile later, tap Skip to go straight to the feed. You can always come back and complete your profile.
5
To view or edit your profile at any time, tap the Profile icon in the navigation bar at the bottom of the screen.
⭐ Reviews & Feed
1
Find a game using Search, then tap Write a Review on the game's detail page.
2
Tap the stars to choose a rating from 1 to 5, then type your written review in the text box below.
3
Tap Submit to post your review. It will appear on the game's page and on the community feed immediately.
4
To read all reviews for a specific game, tap See All Reviews on any game's detail page.
5
To browse the community, tap the Feed tab. Use the toggle at the top to switch between Global (everyone's reviews in real time) and Following (only reviews from gamers you follow).

Technology

How We Built It

GameVault is a full-stack mobile app built with a modern, production-ready tech stack — from a FlutterFlow frontend to a Firebase backend and AI-powered recommendations.

⚡
FlutterFlow
Frontend UI and app logic. FlutterFlow lets us build native Android screens visually while generating clean Dart/Flutter code under the hood.
🔐
Firebase Authentication
Handles all user sign up and sign in with email and password. Secure, scalable, and built into the Firebase ecosystem.
🗄️
Firestore
NoSQL cloud database storing all users, reviews, and favorites. Real-time sync means the feed updates instantly when anyone posts.
🎮
IGDB API
Game data for 500K+ titles — cover art, genres, platforms, developers, and descriptions — fetched via a Twitch Bearer token through a secure Cloud Function proxy.
📊
Firebase Analytics
Tracks user behavior and key metrics including session events, golden path completion, and A/B test performance across all variants.
🧪
Firebase Remote Config
Powers our A/B testing by remotely toggling features per user group — like showing or hiding the onboarding skip button — without a new app release.
🤖
Claude AI
Powers the game recommendation feature. Claude analyzes the titles in your favorites list and returns personalized suggestions instantly.
Research

A/B Testing

We ran three controlled experiments using Firebase Remote Config and Firebase Analytics to measure how design decisions affect real user behavior.

Sign Up / Sign In Screen Layout
Metric: Signup completion rate (Adoption)
Variant A — Control Two separate screens: one for Sign In, one for Sign Up. Users must navigate between them to find the right flow.
Variant B — Test A single combined screen with a toggle to switch between Sign In and Sign Up fields — reducing friction at the very first step.
Profile Setup: Skip Button Visible vs Hidden
Metric: Golden path completion rate (Task Success)
Variant A — Control Skip button hidden during onboarding. Users must complete profile setup before continuing to the app.
Variant B — Test Skip button visible. Users can bypass profile setup and go straight to the feed, completing it later if they choose.
Generic Home Screen vs Personalized Welcome Back Feed
Metric: 7-day and 30-day retention rate
Variant A — Control Standard default home screen shown to all returning users with no personalized content.
Variant B — Test Personalized "Welcome Back" feed showing recent friend activity and the user's recently logged games to give them an immediate reason to re-engage.
