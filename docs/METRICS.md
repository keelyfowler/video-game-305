# Metrics Document
[HEART Framework](https://docs.google.com/presentation/d/1ZGAX6cU6l0ugw-1nD4hcgqJllzeR-fhB8WzF_jOApwY/edit?usp=sharing)
****
Metric 1: NPS - How it will be collected
We can create in-app popups to ask users after engagement that ask "Would you recommend this to a friend?", "Are you enjoying the app?", "What could we improve?", and questions like that.
****
Metric 2: Adoption- Measured using Firebase Authentication signup events.
****
Metric 3: DAU- Collected automatically through Firebase Analytics active users metric.
****
Metric 4: Retention- Measured by tracking the percentage of users who return to the app within 7 days of their initial signup, collected via Firebase Analytics session events.
****
Metric 5: CTR (Click-Through Rate) for Game Search — Measured by tracking the ratio of `search_performed` events to `game_clicked` events via Firebase Analytics custom events. This measures how often users who search for a game actually click through to view a game's info page, which is a key step in the Golden Path.
****
Metric 6: Average Searches Per Active User Per Day — Measured by dividing the total number of `search_performed` events by the number of daily active users, collected via Firebase Analytics. This measures the extent to which users are actively engaging with the core feature of the app.
****
